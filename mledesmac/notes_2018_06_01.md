## Unit test

What is a unit test?

Es el código que nos permite garantizar el correcto funcionamiento de un elemento atómico.

Las pruebas tienen que denotar las reglas de negocio.
Nos ayuda a probar si los cambios relacionados en nuestro còdigo no ha modificado la funcionalidad de la aplicación.

* Escribir código mínimo necesario para que las pruebas pasen.
* no tenemos que pensar cómo resolver el problema, porque ya está automatizado.


##TDD

**Tradicional** -> 90% tiempo dedica a desarrollar sin considerar tiempo para las pruebas
**TDD** -> 80% pruebas unitarias & código
- Comprender la lógica de negocio.
- Plasmar los casos en los cuales se pueden interactuar basado en la lógica de negoci.
- Anticipar los posibles comportamientos del usuario con el sistema.
- Desarrollo el código mínimo necesario para que las pruebas pasen


TDD = Test Driven Development
